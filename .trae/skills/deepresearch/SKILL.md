# 深度研究

## 概述

深度研究技能帮助您在开始实际开发前，全面调查项目实现方案并在GitHub等平台上发现现有参考资料、模板和解决方案。这确保您能够基于现有知识进行构建，而不是重新发明轮子。

## 何时调用

**立即调用此技能当：**

- 您即将开始实现一个新项目
- 您需要研究特定功能的技术实现方法
- 您想找到现有的GitHub仓库或开源解决方案
- 您需要了解特定技术栈的最佳实践
- 项目需求涉及复杂或不熟悉的技术

## 研究流程

### 步骤1：需求分析与领域知识识别

- 分析项目需求并识别关键技术挑战
- 将项目分解为可管理的组件
- 识别项目涉及的专业领域知识（如大气科学、金融等）
- 确定需要研究的核心原理和概念

### 步骤2：原理与知识研究

- 搜集项目相关的基础原理和理论知识
- 研究专业领域的核心概念和方法
- 分析技术实现的底层原理
- 整理领域知识与技术实现的关联

### 步骤3：GitHub与现有方案研究

- 搜索与项目领域相关的现有仓库
- 寻找类似的应用程序或组件
- 根据星标、分支和活跃度评估仓库质量
- 识别可重用的代码、模式或库

### 步骤4：技术可行性分析

- 评估不同实现方案的可行性
- 比较各种技术的优缺点
- 识别潜在的技术风险和缓解策略
- 结合领域知识评估技术方案

### 步骤5：综合参考资料收集与文档生成
- 编制相关GitHub仓库列表
- 整理领域知识和原理资料
- 记录关键发现和见解
- 创建可用资源摘要
- 提供实施建议
- 生成结构化研究文档并保存

### 步骤6：文档保存与管理
- 将研究成果保存为Markdown文档
- 存储在项目目录中便于后续访问
- 生成文档索引以便快速查找
- 确保文档包含完整的研究内容和参考资料

## 使用示例

### 示例1：手机记账应用

**输入：**
"我需要研究如何实现一个手机记账应用，包含支出跟踪、预算管理和AI集成等功能。"

**流程：**

1. 分析需求：移动应用、记账功能、AI集成
2. 研究相关原理：移动应用架构、数据存储方案、AI图像识别
3. 在GitHub上搜索"移动支出跟踪器"、"个人财务管理应用"、"AI记账助手"
4. 评估"Money Manager Expense & Budget"、"Fintech App Templates"等仓库
5. 研究支出分类的AI集成选项
6. 整理发现和建议

### 示例2：大气科学相关项目

**输入：**
"我需要研究如何实现一个基于大气科学知识的AI模型，用于气象数据分析和预测。"

**流程：**

1. 分析需求：大气科学知识集成、AI模型、气象数据分析
2. 研究相关原理：大气动力学、气象数据处理、机器学习模型
3. 在GitHub上搜索"atmospheric science AI"、"weather prediction model"、"meteorological data analysis"
4. 评估相关仓库和开源项目
5. 研究大气科学知识如何与AI模型结合
6. 整理发现和建议

## 输出格式

研究应包括：

### 1. 项目需求与领域知识分析

- 要实现的关键功能
- 识别的技术挑战
- 涉及的专业领域知识（如大气科学等）
- 需要研究的核心原理和概念
- 范围定义

### 2. 原理与知识研究

- 项目相关的基础原理和理论知识
- 专业领域的核心概念和方法
- 技术实现的底层原理
- 领域知识与技术实现的关联

### 3. GitHub与现有方案发现

- 前3-5个相关仓库
- 每个仓库的简要描述
- 使用的关键功能和技术
- 仓库链接

### 4. 技术建议

- 推荐的技术栈
- 实施方法
- 结合领域知识的技术考量
- 要避免的潜在陷阱
- 要遵循的最佳实践

### 5. 实施路线图
- 建议的开发阶段
- 优先功能
- 估计复杂度
- 领域知识集成策略

### 6. 文档保存与管理
- 研究成果保存路径
- 文档命名规范
- 如何在后续对话中引用这些文档
- 文档更新和维护建议

## 最佳实践

- 研究查询要具体
- 关注最近且活跃维护的仓库
- 考虑移动应用的原生和跨平台方法
- 根据您的具体需求评估解决方案
- 彻底记录所有发现以供将来参考
- 定期更新研究文档以反映最新进展
- 使用一致的文档命名规范便于管理
- 在项目启动时参考之前的研究成果
- 建立文档索引系统以提高查找效率

## 结论

深度研究技能提供了一种结构化方法，在开始项目实现前收集技术见解并找到现有解决方案。通过生成并保存详细的研究文档，确保研究成果可以在后续对话和项目开发中被充分利用。这不仅有助于减少开发时间，避免常见陷阱，还能建立知识库，为项目的持续发展提供支持。

研究文档的保存和管理确保了：
- 历史研究成果的可追溯性
- 跨对话的知识连续性
- 项目经验的积累和传承
- 新团队成员的快速上手

通过这种系统化的研究方法，您可以基于经过验证的方法进行构建，同时为项目建立宝贵的知识资产。
